The `tax` binary has three command groups: `return`, `form`, and `node`.

```
tax <command> <subcommand> [flags]
```

## Command groups

| Group | Purpose |
|-------|---------|
| `return` | Create, compute, validate, and export returns |
| `form` | Add, list, update, and delete input documents |
| `node` | Inspect schemas, view dependency graphs, list all nodes |

## Output format

All commands return JSON by default. Pipe to `jq` for pretty printing:

```bash
tax return get --returnId abc-123 | jq .
```

## Workflow

A typical session follows this order:

```bash
# 1. Create
tax return create --year 2025

# 2. Add documents (repeat for each form)
tax form add --returnId ID --node_type TYPE 'JSON'

# 3. Compute
tax return get --returnId ID

# 4. Validate
tax return validate --returnId ID

# 5. Export
tax return export --returnId ID --type mef > return.xml
```
