## `return create`

Create a new tax return for a given year.

```bash
tax return create --year 2025
```

```json
{ "returnId": "abc-123-def-456" }
```

**Flags:**

| Flag | Required | Description |
|------|----------|-------------|
| `--year N` | Yes | Tax year (e.g. 2025) |

Returns are stored locally as files in the current directory under `.tax-returns/`.

---

## `return get`

Execute the full return computation and print all calculated values.

```bash
tax return get --returnId abc-123
```

Runs the dependency graph in topological order and outputs every computed field — AGI, taxable income, total tax, credits, payments, refund or amount owed.

**Flags:**

| Flag | Required | Description |
|------|----------|-------------|
| `--returnId ID` | Yes | Return ID from `return create` |

---

## `return validate`

Run the IRS MeF business rules validation engine against a return.

```bash
tax return validate --returnId abc-123
tax return validate --returnId abc-123 --format json
```

Produces three severity levels:

| Severity | Meaning |
|----------|---------|
| **Error** | Must be corrected before the return can be filed |
| **Warning** | Likely mistake — confirm before filing |
| **Info** | Informational diagnostic |

**Flags:**

| Flag | Required | Description |
|------|----------|-------------|
| `--returnId ID` | Yes | Return ID |
| `--format text\|json` | No | Output format (default: text) |

---

## `return export`

Export the return as IRS MeF XML, ready for submission through a clearinghouse.

```bash
tax return export --returnId abc-123 --type mef > return.xml
```

The XML conforms to IRS 2025v5.2 schema.

**Flags:**

| Flag | Required | Description |
|------|----------|-------------|
| `--returnId ID` | Yes | Return ID |
| `--type mef` | Yes | Export format (currently: mef) |
| `--force` | No | Overwrite if file exists |
