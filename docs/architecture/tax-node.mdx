---
title: "TaxNode"
description: "The single primitive that the entire engine is built on."
---

Every calculation in the engine is expressed as a `TaxNode`. There is exactly one primitive — no special cases, no subclasses for different behaviors.

## The interface

```typescript
abstract class TaxNode<TSchema extends z.ZodTypeAny> {
  abstract readonly nodeType: string;
  abstract readonly inputSchema: TSchema;
  abstract readonly outputNodeTypes: NodeType[];
  abstract compute(input: z.infer<TSchema>): NodeResult;
}

type NodeOutput = {
  nodeType: NodeType;
  input: Record<string, unknown>;
};

type NodeResult = {
  outputs: NodeOutput[];
};
```

A node is three things:
1. **`nodeType`** — a unique string identifier (e.g., `"w2"`, `"f1099int"`)
2. **`inputSchema`** — a Zod schema that validates the node's inputs
3. **`compute()`** — a pure function that takes validated input and returns outputs for downstream nodes

## Schema first

The schema is defined once. Types are inferred from it — never written separately.

```typescript
export const itemSchema = z.object({
  box1_wages: z.number().nonnegative(),
  box2_fed_withheld: z.number().nonnegative(),
  box3_ss_wages: z.number().nonnegative().optional(),
  box13_statutory_employee: z.boolean().optional(),
});

type W2Item = z.infer<typeof itemSchema>;
```

## Type-safe routing

`OutputNodes` enforces at compile time that you can only route to nodes declared in `outputNodeTypes`, and that the input shape matches the target node's schema.

```typescript
class W2Node extends TaxNode<typeof inputSchema> {
  readonly nodeType = "w2";
  readonly inputSchema = inputSchema;
  readonly outputNodes = new OutputNodes([f1040, schedule_c, form4137]);

  compute(input: W2Input): NodeResult {
    const out = new OutputNodes([f1040, schedule_c, form4137]);

    // Compile error if f1040 is not declared, or input doesn't match f1040's schema
    out.add(f1040, {
      line1a_wages: wageTotal(input.items),
      line25a_withheld: withholdingTotal(input.items),
    });

    return { outputs: out.build() };
  }
}
```

## Pure helpers

`compute()` delegates to small pure functions — one per concern.

```typescript
function regularItems(items: W2Items): W2Items {
  return items.filter(item => !item.box13_statutory_employee);
}

function wageTotal(items: W2Items): number {
  return regularItems(items).reduce((sum, item) => sum + item.box1_wages, 0);
}

function withholdingTotal(items: W2Items): number {
  return regularItems(items).reduce((sum, item) => sum + item.box2_fed_withheld, 0);
}
```

## File structure

Every node file follows this order:

```typescript
// 1. Imports
import { z } from "zod";
import { TaxNode, OutputNodes } from "../../../core/types/tax-node.ts";

// 2. Enums (for domain codes)
export enum DistributionCode { Code1 = "1", Code7 = "7", ... }

// 3. Per-item schema
const itemSchema = z.object({ ... });

// 4. Node input schema
const inputSchema = z.object({ items: z.array(itemSchema) });

// 5. Type aliases
type W2Items = z.infer<typeof inputSchema>["items"];

// 6. Pure helper functions
function regularItems(items: W2Items) { ... }
function wageTotal(items: W2Items) { ... }

// 7. Node class
class W2Node extends TaxNode<typeof inputSchema> { ... }

// 8. Singleton export
export const w2 = new W2Node();
```
