Input nodes are the entry points for tax data. Each one corresponds to a real-world tax document or election — a W-2, a 1099, a schedule you fill out, or a special situation.

## How nodes work

When you run `form add`, the data flows into the node's `compute()` function. The node validates it, runs IRS logic, and passes results downstream to the forms that need them.

```
tax form add --returnId abc-123 --node_type w2 '{"box1_wages": 95000}'
                                                        ↓
                                              W2Node.compute()
                                                        ↓
                                              → Form 1040 (wages, withholding)
                                              → Schedule C (if statutory employee)
                                              → Form 4137 (if allocated tips)
```

## Inspect any node

```bash
tax node inspect --node_type w2
```

Shows every field the node accepts, whether it's required, and what it feeds into.

## The `general` node

Every return needs a `general` entry for taxpayer info and filing status:

```bash
tax form add --returnId abc-123 --node_type general \
  '{"filing_status": "single"}'
```

Filing status values: `single`, `married_filing_jointly`, `married_filing_separately`, `head_of_household`, `qualifying_surviving_spouse`
